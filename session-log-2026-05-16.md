## 1. Fitness Tab — `modules/fitness-tab.js` (NEW FILE)

### Why
Macro targets were previously auto-generated by Claude from a one-shot prompt off `settings.currentWeight` + `weightGoal`. No UI to configure or refine — and macro/fitness goals are personal health data, not household. Ryan wanted a proper module with user-level privacy.

### What it does
- 4 goal types: **Lose Weight (-500 cal)**, **Aggressive Cut (-750)**, **Maintain (TDEE)**, **Build Muscle (+250)**
- 5 activity levels: Sedentary (1.2×) → Lightly Active (1.375×) → Active (1.55×) → Very Active (1.725×) → Athlete (1.9×)
- Body metrics: age, sex, weight (kg), height (cm)
- **Mifflin–St Jeor BMR**, then TDEE = BMR × activity multiplier, then goal calorie adjustment.
- Protein scaled by goal: 1.8 g/kg (maintain) → 2.0 (lose) → 2.2 (build) → 2.4 (cut).
- Remaining calories split into carbs/fat per goal ratios.
- Live preview: changing any input updates the targets instantly before saving.
- **Manual override**: toggle to set custom calories/protein/carbs/fat directly (bypasses calc).
- **Share-with-household** toggle (UI built, write path pending Phase 2 of household system).

### Storage
- Profile: `users/<uid>/ml/fitness/profile` (`KEY_PROFILE = "ml:fitness:profile"`) — personal, never shared by default.
- Targets: `users/<uid>/ml/macro/targets` (`KEYS.macroTargets()`) — same key the rest of the app already reads. Existing auto-target generation in `FoodTab.generateMacroTargets()` still works as a fallback for users who haven't set up Fitness yet.

### Wiring
- `index.html`: added `<script src="modules/fitness-tab.js?v=20260516">` after food-tab.
- `app.js` NAV: added `{id: "fitness", l: "FITNESS", c: "var(--color-accent-teal)"}` under the HEALTH section.
- `app.js` tab switch: added `tab === "fitness" && React.createElement(window.FitnessTab, { macroTargets, setMacroTargets, settings })`.

## 2. Meal Photo Scan — `modules/food-tab.js` (MealLogChat)

### Why
Ryan asked: "take a picture of a meal, have it scanned by Opus vision, then break down calories/size/food for each meal and log that automatically — without dictating every component."

### What it does
- New 📷 camera button in the meal log chat input row (next to mic and text input).
- On mobile, `capture="environment"` opens the back camera. On desktop, file picker.
- Image is base64-encoded and sent as an Anthropic vision content block to `claude-sonnet-4-20250514` via the existing `/api/claude` Netlify proxy.
- System prompt now includes both:
  - **KNOWN MEALS** (the user's meal library, top 30 entries with macros) — Claude tries this first.
  - **PANTRY ITEMS IN STOCK** (top 60 pantry items) — Claude is told to prefer these when asking "which lentil?" / "which rice?" etc.
- Behaviour:
  - **Library match** → uses those macros, asks only "1 serving / 1.5 / 2?" → save.
  - **New mixed dish** (e.g. dal & rice) → asks portion questions one at a time, suggesting pantry items as the candidate ingredient.
  - **Simple visible items** (a banana, a shake) → estimates directly.
- The same `parseAndApplyReply` flow handles both text and vision responses, so save vs clarify behaviour is identical.
- Chat now renders photo thumbnails inline (max 180px) for the user's own messages.

### Refactor done along the way
- Extracted `buildSystemPrompt()`, `callClaudeWithMsgs()`, and `parseAndApplyReply()` out of the old `send()` so both the text path and the new image path share one code path.
- Existing text-only flow still uses Haiku 4.5 (cheap, fast); vision uses Sonnet 4 (vision-capable).
- `MealLogChat` now also receives `pantryItems` so the system prompt can include them.

## 3. Auto Food Score — `modules/personal-tab.js` (Evening + FoodScoreCard)

### Why
Manual 0–5 dot slider for "Overall Food Quality" was subjective. Ryan wanted it auto-derived from actual logged meals vs. macro targets, with prompts to log missing meals before showing the final score.

### New helper: `computeFoodScore(mealLog, macroTargets, skippedSlots)`
Defined at top of `personal-tab.js`. Returns:
```js
{
  score: 0–100,
  grade: "A" | "B" | "C" | "D" | "F",
  color: "var(--color-success)" | ...,
  logged: { calories, protein, carbs, fat },
  missingSlots: ["breakfast"],        // slots past their cutoff with no log + not skipped
  pendingSlots: ["dinner"],           // slots still within their window
  slotStatus: { breakfast: "logged" | "skipped" | "missing", ... },
  breakdown: { calPts, protPts, slotPts, bonusPts }
}
```

### Scoring breakdown (out of 100)
- **Calories within ±10% of target** → 40 pts. Linear falloff to 0 at ±30%.
- **Protein ≥90% of target** → 30 pts. Linear falloff to 0 at 50%.
- **Each main slot accounted for (logged or skipped)** → 20 pts split evenly.
- **Bonus: actually logged (not skipped) slots** → up to 10 pts.

Grade thresholds: A ≥90, B ≥75, C ≥60, D ≥40, F < 40.

### Time-aware missing-meal detection
- Breakfast missing after 12pm
- Lunch missing after 4pm
- Dinner missing after 8pm

Slots before their cutoff are "pending" (don't penalise yet). After the cutoff they're "missing" — Evening shows a prompt:

> "No breakfast logged — did you have something?" [LOG] [Skipped]

`LOG` → navigates to Food tab (`onNavigateToFood` passed from App).
`Skipped` → adds slot to `skippedSlots` state; no penalty applied. Stored in `evening.skippedMeals` array.

### Backwards compatibility
The old `foodQuality` (0–5) field is still written — derived from the grade (A=5, B=4, C=3, D=2, F=1) — so historical reads (HistoryBrowser, Sunday reports) keep working. New richer fields added:
- `evening.foodScore` (0–100)
- `evening.foodGrade` ("A"–"F")
- `evening.skippedMeals` (array of slot names)

### UI replacement
Replaced this in Evening:
```js
React.createElement(Card, {
  ch: React.createElement(React.Fragment, null,
    React.createElement(Lbl, { c: "Overall Food Quality" }),
    React.createElement(Dots, { val: fq, set: setFq, col: "..." })
  )
})
```
With:
```js
React.createElement(Card, {
  ch: React.createElement(FoodScoreCard, {
    mealLog: isHistory ? null : mealLog,
    macroTargets, skippedSlots,
    onSkip, onUnskip, onGoLog
  })
})
```

### Edge case
If `macroTargets` is null (Fitness profile not set up yet), the card shows "—" grade and the message "Set up your Fitness profile to enable auto-scoring." — guides the user to the new Fitness tab.
